## **2.2. Use Case Points**

### **UUCW – Взвешенные Use Cases**
- UC-1 — сложный = 15
- Остальные 9 UC — средние = 90

**UUCW = 105**

### **UAW – Взвешенные акторы**
1 актор-человек → 3

**UAW = 3**

### **Таблица технических факторов (TCF)**

**Итого TCF сумма = 35.5**

**TCF – технический коэффициент** 
Формула:
```
TCF = 0.6 + 0.01 × ΣTF = 0.6 + 0.355 = 0.955 ≈ 0.95
```

### **Таблица экологических факторов (ECF)**

**Итого ECF сумма = 14.5**

**ECF – экологический коэффициент**
Формула:
```
ECF = 1.4 - 0.03 × ΣEF = 1.4 - 0.435 = 0.965 ≈ 0.8
```

### **Итоговый расчёт UCP**

```
UCP = (UUCW + UAW) × TCF × ECF
UCP = (105 + 3) × 0.95 × 0.8 = 82
```

### **Трудозатраты**
1 UCP = 20 чел.-часов

```
82 × 20 = 1640 чел.-часов
```

При 160 часов/месяц:
```
1640 / 160 = 10.25 человеко-месяцев
```

## Рекомендации по составу команды и срокам

### Состав команды (оптимальный для реализации полного объёма по UCP ≈1640 час.):
- 1 Tech Lead / Backend (Senior) — 0.5–1.0 FTE
- 1 Backend Developer (Middle) — 1.0 FTE
- 1 Frontend Developer (Middle) — 1.0 FTE
- 1 QA Engineer — 0.5–1.0 FTE
- 1 Product/BA — 0.25–0.5 FTE
- 1 DevOps/Release — 0.25 FTE (частично)

**Ориентировочные сроки:**
- При команде 4–5 человек (микс Middle+Senior): **2–3 месяца** с учётом активной работы, интеграции и тестирования.
- При 2–3 человек: **5–8 месяцев**.

### Специфические требования к ролям
- **Backend:** опыт работы с REST API, авторизацией, загрузкой файлов, транзакциями и очередями уведомлений.
- **Frontend:** опыт работы с компонентными UI, формами, валидацией, списками с пагинацией/фильтрами.
- **QA:** автоматизация API/UI, тестовые сценарии для workflow и граничных случаев.
- **DevOps:** контейнеризация, CI/CD, хранение артефактов, резервное копирование файлов.

**Итог:** проект средней сложности, реалистичный объём ~1640 часов.

## Какие фичи можно отсечь/отложить при необходимости уменьшения масштаба

Если нужно снизить объём работ и сроки, можно перенести на следующий этап или реализовать минимально-жизнеспособные версии (MVP) следующих функциональностей:

1. **Интеграция с внешними календарями (UC-7)** — высока по сложности и интеграциям; её можно отложить.
2. **Шаблоны встреч / сложные сценарии повторений** (повторяемость/исключения) — реализовать позже или в урезанном виде.
3. **Сложная аналитика и экспорт отчётов (UC-9)** — можно сделать базовую статистику, продвинутую аналитику отложить.
4. **Расширенные уровни доступа и тонкая ACL (UC-11 из MeetPlanner)** — на начальном этапе достаточно простых настроек видимости.
5. **Дополнительные методы уведомлений (SMS, сложные push)** — начать с email/push.
